# 2 Basic Theory Teaching: Vibe Coding Getting Started Guide

&nbsp;&nbsp;&nbsp;&nbsp;Our documentation is designed to enable novice developers to quickly learn and develop an AI-native project. Before starting to create, don't rush to write prompts or design interfaces. First, understand a few "keywords," just like learning to read a star chart before setting off.  
&nbsp;&nbsp;&nbsp;&nbsp;You may have heard these terms countless times - **API, deployment, model, database...**
&nbsp;&nbsp;&nbsp;&nbsp;它们听起来冷冰冰，但其实都在描述一件浪漫的事：如何让想法，从你的脑海，变成一个会动的智能世界。



## 2.1 一个 AI 原生应用由哪些部分组成？

&nbsp;&nbsp;&nbsp;&nbsp;想象你正在搭建一个“智能小世界”。这个世界不是由砖头和水泥构成，而是由「对话」「逻辑」「界面」和「数据」组成的。每个部分都有它的角色，就像一个会自己运转的小生态系统：



### 1. AI 模型层（大脑）

&nbsp;&nbsp;&nbsp;&nbsp;AI原生应用的核心是“AI模型”——它负责**理解意图、生成内容、做出决策**。你告诉它你的想法（Prompt），它会思考并给出回应。可以把它理解为：

> “会思考的引擎”，接收语言 → 理解语义 → 生成结果。
<div style="background-color:#e0f7fa; padding:10px; border-radius:8px;">
🌟 <b>例如：</b>

- 你说“帮我生成一个故事的开头”，AI 会构思并写出来。
- 你说“分析这段文本的情绪”，AI 会进行理解与判断。
</div>

&nbsp;&nbsp;&nbsp;&nbsp;它不需要你写算法，而是靠你与它的“语言协作”。  同时不同的模型有不同的性格：有的像作家，擅长讲故事；有的像工程师，擅长推理；有的像画家，擅长创造图像。

&nbsp;&nbsp;&nbsp;&nbsp;当你开始做 AI 应用，你要做的第一件事，就是选择一个“思维伙伴”。  它将决定你的作品，会思考成什么样子。（如果选择适合你的Coding工具，请参照2.2小节）

---

### 2. 应用逻辑层（身体）

&nbsp;&nbsp;&nbsp;&nbsp;如果大脑负责思考，那么身体就是让这个思考动起来的部分。它决定 AI 什么时候思考、如何回应、输出到哪里。在这里，你会定义：

- 输入与输出的路径（比如“用户输入问题 → AI 生成回答 → 显示在聊天框”）  
- 各个组件如何协作（比如“语音识别结果 → 文本理解 → 图片生成”）

&nbsp;&nbsp;&nbsp;&nbsp;这就像为一个智能生命写下“行为规则”。 Vibe Coding 的妙处在于：你不必写复杂的流程图，只需要告诉 AI “我希望它先听、再想、再回答”， AI 就能帮你把流程实现出来。

&nbsp;&nbsp;&nbsp;&nbsp;那么我们就不得不提到 API 了，API 这个词听起来复杂，其实非常简单。它是 “Application Programming Interface” 的缩写，但你可以把它当成：一条连接你与 AI 的电话线。

&nbsp;&nbsp;&nbsp;&nbsp;当你点击“生成故事”按钮，后台就悄悄拨出一个电话：  “嘿，AI，我有个任务给你。”  AI 思考一会儿，然后把结果“说”回来，整个过程就是一次 **API 调用**。你不需要懂算法，也不需要训练模型；你只要学会打电话——而 AI，会永远接听。

---

### 3. 前端层（界面）

&nbsp;&nbsp;&nbsp;&nbsp;这是 AI 世界与人类世界的“窗户”。  无论是网页、手机应用、还是语音助手的界面，它的任务都是让交互变得自然和可感知。可以把它想象成：

> &nbsp;&nbsp;&nbsp;&nbsp;“AI 的外壳”，让人们看到、听到、触摸到 AI 的回应。

&nbsp;&nbsp;&nbsp;&nbsp;在传统开发中，前端意味着 HTML、CSS、JavaScript。  而在 AI 原生开发中，你可以直接用语言告诉系统：

> “我想要一个温柔、简洁、偏米白色调的界面。”

AI 会根据描述生成界面代码与布局。

---

### 4. 数据层（记忆与连接）

&nbsp;&nbsp;&nbsp;&nbsp;每个智能体都需要记忆。AI 原生应用的“记忆”通常来自数据库、外部 API、或上下文缓存。它负责：

- 存储用户输入（比如聊天记录）  
- 保存知识（比如产品信息、个人资料）  
- 连接外部服务（比如天气、地图、支付接口）

&nbsp;&nbsp;&nbsp;&nbsp;它让应用**持续学习与成长**，不再是“一次性对话”，而是“长期陪伴的智能体”。

---

### 5. Vibe 层（氛围与体验）

&nbsp;&nbsp;&nbsp;&nbsp;最后，也是 AI 原生应用最独特的部分——**氛围层**。  它不是代码，不是功能，而是一种“气质”。它决定了这个应用是冷静的、俏皮的，还是温柔的。  你可以让一个聊天机器人像百科全书那样精准、克制；也可以让它像朋友一样轻声回应你的情绪。  功能相同，感觉不同——这就是“Vibe”的力量。

&nbsp;&nbsp;&nbsp;&nbsp;Vibe 层是视觉、语气、节奏与回应方式共同塑造的“灵魂”。  它让一个 AI 应用不只是能用，而是**好用、想用、甚至让人共鸣**。

## 2.2 当创造不再需要从零开始——Vibe Coding 的工具

&nbsp;&nbsp;&nbsp;&nbsp;下面把常见工具按「用途」分组，给出简短说明、适合对象与使用建议。为避免信息过载，我把每类选出 2–4 个代表性工具，标注它们适合的新手/进阶/团队场景。



### 1. AI 对话式 IDE / 智能编辑器

&nbsp;&nbsp;&nbsp;&nbsp;**适合人群：** 想写代码但要强大 AI 帮忙的人。

&nbsp;&nbsp;&nbsp;&nbsp;这些工具把 AI 嵌入到代码编辑器里，不止补全代码，还能理解项目上下文、生成文件、解释或修复错误。

- **Trae（第四实战部分示例）** ：类似“加强版 VS Code”，集成聊天助手与“建造者模式”，能拆解任务并执行。适合希望把 AI 当“工程师”来用的团队或个人。注意：近期围绕隐私与数据上报的讨论，需要根据项目性质谨慎评估是否上传敏感代码到云端。
- **Replit Ghostwriter** ：在线 IDE，适合教学和快速原型，能即时运行代码并通过聊天解释报错，入门零门槛。  
- **Cursor / GitHub Copilot** ： 更适合有一定编程基础的开发者，能在本地或云端提供上下文感知的代码建议（其中 Cursor 强调项目级上下文和无缝 GitHub 集成）。

> 小提示：如果你是完全新手，先用 Replit Ghostwriter 之类能即刻运行并给出错误说明的在线 IDE，会更省心；如果你要做团队或较大项目，可以考虑 Cursor / Trae，但要注意隐私与公司政策。

---

### 2. AI 网站 / 界面生成器

&nbsp;&nbsp;&nbsp;&nbsp;**适合人群：** 不想写大量代码但想做网页/小型项目的用户。

- **Framer AI** ：可通过对话快速生成页面结构并继续在所见即所得编辑器中调整，适合视觉导向的初学者与设计师。  
- **Z.AI / v0.app** ： 更偏向「从愿景到全栈原型」，能帮助生成前端和部分后端代码片段，适合想快速实验交互的初创者。  

> 小提示：想快速做“上线展示”优先选 Framer；想把原型进一步演进为可部署的应用可试 v0.app。

---

### 3. Prompt 设计与对话优化

&nbsp;&nbsp;&nbsp;&nbsp;**适合人群：** 想学会与 AI “对话式编程”的任何人。
&nbsp;&nbsp;&nbsp;&nbsp;Prompt（提示词）是 Vibe Coding 的语言核心。好的 Prompt 能让模型理解你的意图、语气与逻辑。

- **ChatGPT / Claude / Gemini / Perplexity**：主流的对话与文本生成模型，各有侧重（价格、隐私、输出风格）。
- **Model orchestration / SDK**：LangChain、Flowise（用于把多步 Prompt 和模型调用组织成流程）——适合进阶用户把多模型任务串联成工作流。

> 小提示：学习 Prompt 先在 ChatGPT / Claude 上多练手，再用 LangChain 把成熟的 Prompt 流程工程化

---

### 4. 视觉与媒体生成

&nbsp;&nbsp;&nbsp;&nbsp;**适合人群：** 为应用增加氛围和视觉内容的创作者。
&nbsp;&nbsp;&nbsp;&nbsp;Vibe Coding 不只是逻辑，也关乎“视觉气息”。这些工具帮助你生成插画、动画、视频或 UI 素材。

- **Midjourney、DALL·E 3、Runway**:生成插图、背景、界面素材；Runway 还可做视频/动效。
- **Pika、Canva AI**:更面向设计师，便于合成 UI 元素和快速原型。
> 小提示：如果你要在网页中展示 AI 生成的图片，先在设计工具中产出一套风格统一的素材，再交给前端调用。

---

### 5. 部署与托管

&nbsp;&nbsp;&nbsp;&nbsp;**适合人群：** 想让项目“真正上线”的开发者。
&nbsp;&nbsp;&nbsp;&nbsp;“部署”是一个关键的转折点。 它意味着你的创作不再只是草稿，而是要与世界见面。

- **Vercel / Netlify / Render / Zeabur** :一键部署网页或静态项目，让应用获得真实网址，非常适合小型作品展示或课程项目
- **Docker / Kubernete**s:更高级的部署方式，用于需要控制服务器环境或运行多个模型的场景，适合团队协作与正式生产环境。

## 2.4 Vibe Coding 对开发者角色的重新定义

&nbsp;&nbsp;&nbsp;&nbsp;在传统的软件开发中，“开发者”往往意味着要掌握复杂的编程语言、框架与算法。一个项目通常需要多个角色协作：前端开发负责界面、后端开发负责逻辑、算法工程师负责模型、设计师负责交互。这种模式效率高、分工明确，但也让许多想“自己做点东西”的人被挡在门外。
&nbsp;&nbsp;&nbsp;&nbsp;而 Vibe Coding 的出现，正在悄悄改变这一切。在这个新的开发范式中，AI 不再只是一个工具，而是一位始终参与在场的“协作者”。开发者的角色，正在从“编写代码的人”，转变为“与 AI 一起构思、实现想法的人”。这种转变有几个显著的特征：

1. **从“写代码”到“组织智能”**  
&nbsp;&nbsp;&nbsp;&nbsp;Vibe Coding 不要求每个人都能熟练掌握一门语言。你需要学习的，是如何让 AI 理解你的意图、如何描述需求、如何用 prompt 引导模型生成你想要的结果。开发的过程更像是一种“设计思维”与“逻辑表达”的结合——你用自然语言去规划逻辑，AI 帮你完成代码实现。
2. **从“技术执行者”到“创意策划者”**  
&nbsp;&nbsp;&nbsp;&nbsp;传统开发更像是在“实现他人的想法”；而在 Vibe Coding 中，你有机会直接构思、验证、迭代自己的创意。 AI 能帮你快速搭建原型、生成接口、甚至优化交互。 你不再需要等待设计图或接口文档，而是能立刻“看到想法成形”。 这让开发者的工作变得更加创造性，也更加贴近产品思维。
3. **从“单一分工”到“混合型能力”**  
&nbsp;&nbsp;&nbsp;&nbsp;Vibe Coding 模糊了传统的岗位边界。一个人可能同时是产品经理、开发者、设计师、甚至内容创作者。 AI 工具让这种“跨界”成为可能：你可以用一个平台完成逻辑编写、UI 设计和模型调用。 因此，Vibe Coding 更强调“解决问题的整体能力”，而不是单一的技术熟练度。
4. **从“独立开发”到“协同共创”**
&nbsp;&nbsp;&nbsp;&nbsp;AI 原生开发环境本身也鼓励协作。通过云端平台和可共享的项目模板，不同背景的人可以在同一个空间中协同构建。这意味着未来的“开发团队”，不一定全是程序员——它可以是设计师 + AI 研究者 + 内容创作者 + 初学者共同组成的创作小组。

&nbsp;&nbsp;&nbsp;&nbsp;从以上信息中你或许可以发现，Vibe Coding 的核心价值并不是让所有人都成为程序员，而是让“思考、表达、实现”之间的距离变得前所未有地短。你可以用语言定义逻辑，用想法生成交互，用直觉设计体验。在这个意义上，Vibe Coding 正在重新定义“开发者”这一身份——它更像是一种新的创造力形态，而不是一种技术职位。
